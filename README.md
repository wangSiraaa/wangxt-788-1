# 高校自习室座位热力图系统

一个基于 React + TypeScript + Tailwind CSS 的高校自习室座位预约与管理系统。

## 功能特性

### 学生端
- 🗺️ **热力图展示**：直观的座位分布热力图，不同颜色表示不同状态
- 🔍 **多维度筛选**：按楼栋、楼层、日期、时段、座位状态、区域筛选
- 📅 **在线预约**：选择座位后一键预约
- ⚠️ **冲突检测**：自动检测预约冲突（已占座位、同时段重复预约等）
- 🔄 **替代楼层推荐**：预约冲突时智能推荐同楼或相邻楼层的可用座位
- 📊 **占用统计**：实时查看楼层座位占用率和统计数据

### 管理员端
- 🚫 **区域关闭**：可按区域批量关闭座位（如设备维护、活动占用等）
- 🔓 **区域管理**：管理已关闭区域，支持重新开放

### 保洁端
- 🧹 **清洁标记**：一键标记座位清洁状态
- 👁️ **状态展示**：清洁中座位学生不可预约，但仍可查看

## 业务规则（可复现）

1. **已占座位不可再次选择**：状态为 occupied 的座位学生无法点击预约
2. **同一时段单一座位**：同一学生在同一时段只能保留一个有效预约
3. **智能替代推荐**：预约冲突时，优先推荐同楼栋相邻楼层，其次按可用座位数排序
4. **清洁中座位限制**：清洁中座位仅展示不可预约，由保洁人员切换状态
5. **关闭区域隔离**：管理员关闭的区域内所有座位自动标记为不可用

## 技术栈

- **框架**：React 18 + TypeScript
- **构建工具**：Vite 5
- **样式方案**：Tailwind CSS 3
- **容器化**：Docker + Nginx

## 快速开始

### 方式一：本地开发

```bash
# 安装依赖
npm install

# 启动开发服务器
npm run dev
```

访问 http://localhost:3000 查看应用。

### 方式二：Docker 部署

```bash
# 构建并启动容器
docker-compose up -d

# 或单独构建镜像
docker build -t study-room-heatmap .
docker run -p 3000:80 study-room-heatmap
```

访问 http://localhost:3000 查看应用。

### 方式三：生产构建

```bash
# 构建生产版本
npm run build

# 预览生产构建
npm run preview
```

## 运行检查脚本

项目包含 E2E 业务逻辑检查脚本，验证核心规则：

```bash
npm run test:e2e
```

脚本会执行以下检查：
1. ✅ 选择已占座位，断言冲突反馈正确
2. ✅ 制造同一时段重复预约，验证替代楼层列表出现
3. ✅ 清洁中座位只能展示不能预约
4. ✅ 已关闭座位不可预约
5. ✅ 可用座位无冲突

## 角色切换说明

系统内置三种角色，可在顶部导航栏切换：

- **学生**：查看热力图、筛选座位、预约座位、查看预约记录
- **楼层管理员**：管理关闭区域、开放/关闭座位区域
- **保洁人员**：标记座位清洁状态、管理清洁中的座位

## 移动端支持

系统采用响应式设计，支持：
- 手机端单列布局
- 平板端双列布局
- 桌面端三列布局
- 触摸友好的交互设计

## 本地数据说明

所有数据均存储在浏览器内存中，包含：
- 2 栋教学楼（图书馆主楼、教学楼A座）
- 5 个楼层自习室
- 约 250 个座位（含不同状态）
- 5 个预约时段
- 4 个测试用户（2学生、1管理员、1保洁）
- 示例预约记录和关闭区域

刷新页面后数据会重置为初始状态。
